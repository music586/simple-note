# 斜杠菜单选中态修复设计

## 问题与根因

斜杠菜单的键盘处理会更新选中索引，但选中样式引用了未定义的 CSS 变量
`--bg-hover`。浏览器丢弃该背景声明，导致默认选中项和上下键切换都没有可见反馈，
用户因而感知为“没有选中状态、上下键无交互”。

当前按键分发已通过行为测试：菜单属于当前编辑器且存在结果时，`Up` 和 `Down`
分别调用 `moveSlashCommandSelection(-1)` 与 `moveSlashCommandSelection(1)`。
本次不重写按键架构，只修复视觉反馈并补齐鼠标与状态回归测试。

## 交互设计

- 菜单打开时默认选中第一项。
- 选中项使用轻量淡紫背景和轻微文字提亮，不增加左侧指示条或位移动画。
- `Up` / `Down` 循环切换选中项，切换后自动滚动到可见区域。
- 鼠标移入选项时同步更新选中索引，使鼠标、键盘和 Enter 指向同一项。
- Enter 插入当前选中项；鼠标按下继续保持 CodeMirror 焦点并插入该项。
- 没有筛选结果时，上下键继续返回 `CodeMirror.Pass`。
- 中文输入法组合输入的键盘优先级保持不变。

## 实现边界

- 在明暗主题变量中定义菜单选中背景与文字颜色，禁止引用未定义变量。
- `aria-selected='true'` 同时驱动视觉样式和无障碍活动项状态。
- 菜单选项监听 `mouseenter`，仅在索引确实变化时更新状态并重绘。
- 将循环索引计算提取为纯函数，供 DOM 逻辑与测试共同使用。
- 不修改 Markdown 命令、查询筛选、插入文本、IME、代码块或结构键行为。

## 验证

- 纯函数测试：首项向上循环到末项、末项向下循环到首项、普通移动、零项安全。
- 源码/CSS 回归：选中样式使用已定义变量，鼠标移入会同步选择。
- 现有键盘测试：菜单归属、结果为空、Up/Down 调用和 Enter 行为保持通过。
- 完整 `npm test`、JavaScript 语法检查与 `git diff --check`。
- Electron 人工检查：暗色/亮色主题中默认选中可见，上下键每次移动一项，鼠标移入
  与 Enter 插入一致，长菜单自动滚动。
