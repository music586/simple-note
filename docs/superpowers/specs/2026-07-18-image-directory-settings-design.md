# 图片文件目录设置设计

## 目标

在 macOS“简记”应用菜单中新增“设置…”入口。点击后在主窗口中央打开设置弹窗，允许用户通过系统目录选择器配置一个对所有笔记生效的图片文件目录。

## 交互设计

- “简记”菜单新增“设置…”。
- 菜单点击后向当前主窗口发送打开设置弹窗事件。
- 设置弹窗居中显示，沿用应用现有弹窗的颜色、圆角、遮罩和中文界面。
- 弹窗包含：
  - 标题“设置”；
  - 设置项“图片文件目录”；
  - 当前生效目录，只读展示并允许长路径换行或省略；
  - “选择目录”按钮，打开系统目录选择器；
  - “恢复默认”按钮，清除全局自定义目录；
  - “完成”按钮，关闭弹窗。
- 不提供路径手动输入。
- 点击遮罩或按 Escape 可关闭弹窗，不改变已保存配置。

## 目录规则

- 未设置自定义目录时，沿用现有规则：图片保存到当前笔记库根目录下的 `assets` 文件夹。
- 弹窗默认展示当前笔记库对应的默认图片目录，不要求先打开笔记。
- 选择自定义目录后，该目录立即保存到 `config.json`，对所有笔记生效。
- 恢复默认后删除 `config.json` 中的图片目录配置，后续图片重新使用各笔记旁的 `assets` 文件夹。
- 已有图片不复制、不移动、不改写 Markdown。
- 后续通过剪贴板粘贴或下载保存的图片写入当前生效目录。
- Markdown 图片地址仍以当前笔记目录为基准生成相对路径；无法生成相对路径时仍使用 Node.js 的跨目录相对路径形式。

## 组件与数据流

### 主进程

- 菜单通过 IPC 通知渲染进程打开设置弹窗。
- 新增读取图片目录状态的 IPC：返回自定义目录、当前默认目录和最终生效目录。
- 新增选择图片目录的 IPC：使用 `dialog.showOpenDialog` 且只允许选择文件夹；选择成功后写入配置。
- 新增恢复默认目录的 IPC：删除自定义图片目录配置。
- 图片粘贴保存逻辑统一调用目录解析函数，避免 UI 展示路径与实际保存路径不一致。
- 所有传入的笔记路径继续执行既有路径校验。

### 渲染进程

- 接收菜单事件后获取目录状态、填充弹窗并显示。
- 选择或恢复默认成功后刷新当前路径展示。
- 用户取消系统目录选择时保持当前设置不变。
- IPC 失败时在弹窗内显示中文错误，不关闭弹窗。

## 错误处理

- 自定义目录丢失或无法访问时，保存图片操作返回明确错误，不静默回退，避免图片保存到用户未预期的位置。
- 目录选择取消不视为错误。
- 配置文件读取失败沿用当前配置容错策略；非法的图片目录值不作为有效自定义目录。

## 测试范围

- 应用菜单包含“设置…”并发送正确事件。
- 默认目录为当前笔记库根目录下的 `assets`。
- 自定义目录优先于默认目录并全局生效。
- 恢复默认会清除自定义目录。
- 粘贴图片使用统一解析后的目录并生成相对 Markdown 路径。
- 设置弹窗包含当前目录、“选择目录”“恢复默认”和“完成”，且没有可编辑路径输入框。
- 取消选择、无当前笔记和 IPC 错误状态有对应行为。
