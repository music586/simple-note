# Markdown 列表配色与缩进设计

## 目标

减少 Markdown 列表引入的额外颜色，并让编辑区域的 Tab 缩进宽度约等于两个中文字符。

## 范围

- 编辑区域预渲染的有序列表和无序列表标记
- 阅读预览中的有序列表和无序列表标记
- 编辑区域 CodeMirror 的缩进配置
- 任务列表复选框沿用同一列表颜色变量

不调整侧栏列表、弹窗列表、正文排版、列表间距或 Markdown 解析逻辑。

## 设计

### 配色

列表颜色变量 `--md-list` 不再定义独立的绿色，改为引用正文颜色变量
`--md-body`。深色和浅色主题均通过各自的 `--md-body` 自动获得合适的对比度。

这会统一以下元素的颜色：

- 编辑区域预渲染的序号和项目符号
- 阅读预览的序号和项目符号
- 任务列表复选框的强调色

### 缩进

CodeMirror 的 `indentUnit` 和 `tabSize` 从 2 调整为 4。编辑区正文中的半角字符宽度
约为中文全角字符的一半，因此 4 个半角空格约等于两个中文字符的视觉宽度。

Tab 操作仍生成空格缩进，不引入真实制表符，以保持 Markdown 文件在其他编辑器中的表现稳定。

## 验证

- 执行 `node --check renderer.js`
- 执行 `git diff --check`
- 检查深色和浅色主题变量均不再使用独立列表绿色
- 检查 CodeMirror 的 `indentUnit` 与 `tabSize` 均为 4

渲染进程修改后刷新窗口即可验证；若刷新后配置未重新初始化，则重启应用。
