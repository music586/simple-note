# Markdown 行级结构输入体验设计

## 目标

为“简记”的 Markdown 编辑器增加低学习成本、全键盘可操作的行级结构输入体验，
同时保留原始 Markdown 文本和熟练用户的直接输入习惯。

第一阶段仅覆盖标题、无序列表、有序列表、任务列表和引用。不引入所见即所得
块编辑，不改变现有实时预览、自动保存、双栏编辑、表格和代码块增强能力。

## 用户与成功标准

目标用户同时包括：

- 不熟悉 Markdown、希望通过中文菜单选择结构的普通笔记用户。
- 熟悉 Markdown、依赖快捷输入和键盘操作的高频用户。

成功标准：

- 新用户无需记忆语法即可在空行创建所有目标结构。
- 熟练用户可以直接输入 Markdown，且原有输入不受菜单干扰。
- 列表、任务和引用可以连续录入、缩进与自然退出。
- 所有自动编辑都能通过一次撤销完整恢复。
- 左右编辑器行为一致，代码块内容不会被结构自动化修改。

## 交互入口

仅当光标位于空行行首时，输入 `/` 打开中文结构菜单。正文、URL、文件路径、
非空行以及代码块中的 `/` 均按普通字符处理。

菜单锚定在光标下方，空间不足时显示在上方，并始终限制在当前编辑面板内。
滚动、切换笔记、编辑器失焦或进入代码块时关闭菜单。

菜单支持：

- 继续输入以筛选命令。
- `ArrowUp` / `ArrowDown` 移动选中项。
- `Enter` 插入选中结构。
- `Escape` 关闭菜单并保留当前输入。
- 鼠标点击命令。

首屏依次展示一级标题、二级标题、无序列表、有序列表、任务列表和引用。
标题命令通过筛选“标题”展开 H1–H6，避免菜单过长。每项同时显示中文名称和
Markdown 标记，使菜单兼具操作入口和轻量学习提示。

筛选匹配中文名称、拼音首字母和 Markdown 标记。例如 `/rw`、`/任务` 和 `/-`
均可定位目标结构。无匹配时显示“没有匹配的结构”，不自动删除用户输入。

## 插入与键盘行为

### 插入

选择命令后，编辑器删除 `/` 及其筛选文字，插入结构前缀，并将光标定位到内容
输入位置。例如任务列表插入 `- [ ] `。

### 回车

- 标题：创建普通空行，不延续标题前缀。
- 无序列表：延续当前缩进并插入 `- `。
- 有序列表：延续当前缩进并将当前编号加一。
- 任务列表：插入新的未完成项 `- [ ] `，不继承上一项的完成状态。
- 引用：延续现有引用层级。

有序列表仅计算新项目编号，不主动重排文档中已有编号，避免隐式改写正文。

### 退出结构

在空列表项、空任务项或空引用行按 `Enter`，移除当前标记并回到普通空行。
因此用户完成最后一项后连续按两次 `Enter` 即可退出结构。

### 缩进

光标位于列表或任务列表中时：

- `Tab` 增加一级缩进。
- `Shift+Tab` 减少一级缩进。

每一级使用现有编辑器的两个空格设置。非列表区域保留 CodeMirror 原有 Tab 行为。

### 删除结构

光标位于结构内容起点时按 `Backspace`：

- 嵌套列表或任务列表优先减少一级缩进。
- 顶层结构移除 Markdown 标记并保留文字。
- 普通文本继续使用 CodeMirror 默认删除行为。

## 技术设计

### 行上下文分析

新增纯逻辑的 `markdownLineContext` 单元，输入编辑器文本和光标位置，输出：

- 当前行内容、缩进和光标列。
- 是否处于围栏代码块。
- 是否满足空行行首触发条件。
- 当前结构类型、标记范围、内容起点、层级和有序列表编号。

解析结果不确定时返回“普通文本”，由 CodeMirror 执行默认行为。该单元不直接
操作 DOM 或编辑器，以便独立测试。

### 结构命令定义

`markdownStructureCommands` 集中定义 H1–H6、无序列表、有序列表、任务列表和
引用的名称、筛选关键词、视觉标记和插入前缀。菜单展示和键盘插入共用同一份定义。

### 斜杠菜单

页面只创建一个 `slashCommandMenu` 实例。它记录当前激活的编辑器适配器和菜单
状态，在左右编辑器间复用，但不共享文档内容或光标状态。

菜单打开时，方向键、Enter 和 Escape 优先由菜单处理；菜单关闭后，这些按键交还
给结构编辑或 CodeMirror 默认行为。中文输入法处于 composition 状态时不执行菜单
筛选、确认或结构快捷操作。

### 结构编辑事务

Enter、Tab、Shift+Tab 和 Backspace 的结构变换通过统一的结构编辑函数完成。
每次变换使用单个 CodeMirror `operation`，保证：

- 一次操作对应一条撤销记录。
- 现有 change 监听器仍能触发自动保存、编辑器装饰和预览刷新。
- 左右编辑器使用相同逻辑，不复制实现。

现有代码块 Enter 处理优先于普通结构处理；围栏代码块内部始终回退到默认编辑行为。

## 状态与数据流

1. CodeMirror 接收输入或按键。
2. 行上下文分析器读取当前行及必要的前序围栏信息。
3. 空行行首输入 `/` 时，菜单绑定当前编辑器并按光标坐标打开。
4. 菜单选择转换为结构命令，在一次编辑事务中替换查询文本。
5. Enter、缩进或删除按键根据行上下文生成最小文本变更。
6. CodeMirror 发出原有 change 事件，继续执行保存、装饰和预览流程。

菜单状态只存在于渲染进程内，不写入笔记或 `localStorage`。

## 边界与错误处理

- 无当前笔记时不打开菜单。
- 代码块、正文中间、URL 和路径中的 `/` 不触发菜单。
- 菜单定位信息不可用时关闭菜单，字符输入保持不变。
- 行结构不符合已知模式时执行默认 CodeMirror 行为。
- 切换笔记、切换活动编辑器或关闭右栏时立即清理菜单状态。
- 自动操作只修改当前行或紧邻的新行，不扫描并重写整篇文档。

## 可访问性与视觉

- 菜单使用现有明暗主题变量，不引入独立配色体系。
- 选中项使用当前主题强调色，其他装饰保持克制。
- 菜单使用合适的 listbox/option 语义及 `aria-activedescendant`。
- 键盘焦点保留在 CodeMirror 输入区，屏幕阅读器可获知菜单开关和当前选项。
- 菜单不得遮挡当前输入行；必要时翻转显示方向。

## 测试设计

使用 Node 内置 `node:test` 测试纯逻辑，不增加第三方测试依赖。覆盖：

- 所有结构的识别、插入、延续、退出与撤销所需文本变换。
- 有序列表编号递增、不同缩进层级和非标准行的安全回退。
- 任务列表始终创建未完成项。
- Tab、Shift+Tab 和 Backspace 的顶层与嵌套行为。
- 围栏代码块内禁用触发和自动化。
- 正文、URL、路径和非空行中的 `/` 不触发。

Electron 手动验收覆盖：

- 菜单鼠标操作、键盘导航、筛选、空结果和 Escape。
- 菜单在编辑器边缘翻转且不越界。
- 中文输入法 composition 期间不误触发。
- 左右编辑器切换、笔记切换和右栏关闭。
- 自动保存、实时预览、代码块语言菜单、表格与图片粘贴没有回归。
- 明暗主题和键盘焦点样式清晰可见。

## 非目标

第一阶段不包含：

- 隐藏 Markdown 标记或所见即所得块编辑。
- 行内格式菜单（粗体、斜体、链接、行内代码）。
- 表格、代码块、脚注和目录的新输入能力。
- 自动重排整篇有序列表编号。
- 多行选择批量转换结构。
